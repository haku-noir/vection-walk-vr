視野反転は　Unity　を用いて実現しています．この資料は　Unity　の大まかな設定に関する資料です．バージョンは 2021.3.24f1 で作成しています．

UnityのVRの開発環境に関しては，インターネットを参照してください．情報が最も多いのは[こちらのサイト](https://tech.framesynthesis.co.jp/unity/metaquest/ "Unity + Meta Quest開発メモ")です．渡邊さんとまとめた[GoogleDocument](https://docs.google.com/document/d/13n0ONjqnUoCllXeg8MT_iWcLPR3zFuggDSr-rQ2jIGQ/edit?usp=sharing "Oculusアプリ化マニュアルstacklist")も参考にしてください．

以下では，これらに書かれていないことに関しての補足説明を書いておきます．

## OVRCameraRigプレハブ

VR 実装の要となるプレハブです．Oculus Integrationを入れると Assets > Oculus > VR > Prefabs フォルダ内に入っています．主要な設定項目について説明しておきます．

#### OVRCameraRig.cs

- Use Per Eye Cameras: 両眼視差の有無を指定します．False のとき，両眼の映像は同一になります．
- Disable Eye Anchor Cameras: カメラを無効化します．映像は別でトラッキングだけしたいときに True にするといいでしょう．

#### OVRManager.cs

- Target Devices: 使用する HMD のチェックが外れていないか確認しましょう．
- Monoscopic: Use Per Eye Cameras の否定とほぼ同じだと思います．
- Tracking Origin Type: Eye Level であれば，プレハブを置いた位置からの映像が映ります．細かいことは調べてもらった方がわかりやすいかと思います．
  
## Playerプレハブ

今回の視野反転の要となるプレハブです．Assets > Prefabs フォルダに入っています．OVRPlayerControllerプレハブを用いて作成していますが，ついている Character Controller や OVR Player Controller スクリプトは余計な動作をさせる元となるので非アクティブにしておきましょう．（主にキーボードやコントローラ操作で動かしたい場合に使用するものです．）メインはやはり OVRCameraRig で，デフォルトのものにいろいろと追加しています．

### ヒエラルキー構造

ヒエラルキーの構造は次のようになっています．（対称性のある部分や関係のない部分は省略しています．）

- OVRCameraRig
  - TrackingSpace
    - LeftEyeAnchor
    - CenterEyeAnchor
      - CenterEyeCapture
      - CenterCanvas
        - CenterRawImage
    - RightEyeAnchor
    - TrackerAnchor
    - LeftHandAnchor
      - LeftControllerAnchor
      - LeftControllerInHandAnchor
      - OVRControllerPrefab (非アクティブ)
      - OculusTouchForQuestAndRiftS_Left
    - RightHandAnchor
    - LeftHandAnchorDetached
    - RightHandAnchorDetached
  
HMD のトラッキングデータ（Position, Rotation）は CenterEyeAnchor に反映されます．すなわち，HMD の動きと CenterEyeAnchor の動きが同期することになります．そのため，これらの子オブジェクトも HMD に同期することになります．（トラッキング誤差は多くても数cmだと思われます．）また，この頭部と同期して動く CenterEyeAnchor のカメラ映像が，HMD に出力されます．

### 視野反転の動作原理

世界の映像を撮影するカメラは， CenterEyeAnchor ではなく，CenterEyeCapture です．このカメラの映像は，「ターゲットテクスチャ (Target Texture) 」に設定された CenterEye というテクスチャに出力されます．（このテクスチャのサイズは HMD 接続時のカメラのアスペクト比に合わせておきます．）このテクスチャを CenterCanvas 内の CenterRawImage のテクスチャに設定することで，カメラ映像が CenterCanvas によって UI として映し出されます．この UI を CenterEyeAnchor で撮影することで，HMD に出力しています．

#### 注意すべき設定

この動作原理を実現する上で Unity 初心者が見落としがちな設定について述べておきます．

- カリングマスク (Culling Mask): カメラが撮影するレイヤーを指定します．オブジェクトのレイヤーはインスペクターの右上で設定できます． UI が CenterEyeCapture に映ると困るので， "UI" のチェックを外しておきます．逆に， CenterEyeAnchor は UI 以外が映ると困るので， "UI" 以外のチェックを全て外しておきます．
- ターゲットアイ (target Eye): HMD に出力するか否かの設定です．CenterEyeCapture のターゲットアイは「なし」にしておきます．

### 手のトラッキング

手（コントローラ）のトラッキングは， LeftHandAnchor および RightHandAnchor に反映されます．このままでは何も見えないので，見えるようにするためには別でつけてあげましょう．今回は OculusTouchForQuestAndRiftS_Left などがついています．これは Assets > Oculus > VR > Meshes > OculusTouchForQuestAndRiftS フォルダにあります．

### その他

OVRCameraRig の初期のローカル座標が (0, 1.5, 0.15) となっていますが，これが HMD を被った時の最初の位置です．Player オブジェクトの位置は体の垂直軸と地面との交点だと思ってください．そこに対する HMD の相対位置は，およそ上に 1.5m，前に 0.15m であるということでこのような設定になっています．（目の高さは比較的リアリティに重要ですが，前後方向の細かい値は正直どうでもいいです．） CenterEyeAnchor の Transform は常に HMD のトラッキングに依存しますので，そちらで設定しても意味はありません．

## スクリプトに関して

今回の Unity プロジェクトには非常に多くのスクリプトがついています．主要なスクリプトに関してはスクリプト内にいろいろと説明を書いていますので，そちらを参考にしてください．

### Quest にビルドしたアプリで収集したデータの保存先

スクリプト内の関数で CSV ファイルとしてデータを保存することが多いですが，アプリ化した際の保存先はスクリプト内では "Application.persistentDataPath" として示されています．このパスは，実際には "storage/emulated/0/Android/data/AppPackageName/files" のようになっています．（"AppPackageName" は， Unity 内の Project Settings > プレイヤー (Player) > プロダクト名 (Product Name) で指定できます．）ここに保存されたデータの取り出しは， SideQuest という非公式アプリを使うことを個人的には推奨します． PC につないだだけでは上手く内部ストレージにエクスプローラーから接続できないことがあります．

## シーン一覧

- 8figure: 8の字歩行実験
- AroundSphere: 周囲に生成される球を視野中央に収めたり，タッチしたりする実験
- LogRecording: トラッキングのログ確認用
- MirrorImage: 回転の挙動の確認用
- OutdoorReversion: 屋外環境．適当な実験環境に．円柱に当たる実験もこちら．コントローラで操作．
- PassThrough: パススルー映像を Unity 内で見ようとした形跡．
- RandomDotPixels: ピクセル単位でのランダムドットの提示（GitHub にもあり）
- RandomDots: ランダムドットの提示（KIM さんの改良）
- RoomReversion: 屋内環境．適当な実験環境に．
- SphereLauncher: 奥からこちらにとんでくる球にタッチする実験．
- Tracking: トラッキング実験（未使用）
- Walking: 直進歩行実験
- WalkingCourse: 一般歩行実験

### 実行方法

シーン1つ1つについて説明していてはきりがないので，多くのものに共通のことを説明しておきます．細かい動作は各スクリプトを確認するか，個人的に連絡をお願いします．

まず，ランダムドットに関しては GitHub に詳しい説明があるのでそちらを確認してください．

視野反転実験のものにおいては，まず大抵の場合世界の時間が停止した状態でスタートします．場合によっては視野に色がついている（真っ黒の場合を含む）かと思います．
キーボードおよび Touch コントローラでの操作は次のようになっています．

- Oキー/Bボタン: 実験の中止・開始（世界の時間の停止・視野の色もこちら）
- Lキー/Yボタン: 視野左右反転
- Uキー/Xボタン: 視野上下反転
- Fキー/左中指トリガー: 視野上下反転
  
実験の内容に関しては Overleaf を参照してください．行った実験は付録に大量についています．データの解析は甘々です．
例えば，直進歩行実験であれば次のようにします．HMD を使わないなら2番は飛ばしてください．

1. Walking シーンを開き，エディターを実行．Quest ならアプリを実行．このとき視野は真っ暗なはず．
2. 実世界で開始地点に立ち，その状態で正面を向いて視点をリセットする．エディターからなら Oculus ボタンを押して，「視点をリセット」から．アプリなら Oculus ボタン長押し．
3. 視野反転を上記に従って好きに設定．（後からでも変更可能．）
4. 上記に従って実験を開始．OキーやBボタンを押せば視野が真っ暗でなくなる．
